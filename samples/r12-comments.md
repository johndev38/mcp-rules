# R12 - Commentaires des sections

## Objectif

Chaque section doit être commentée de manière claire, structurée et lisible.

## Règles obligatoires

Un fichier conforme doit contenir :

### 1. Commentaire d'entête (module)

- Présent au début du bloc FUNCTION
- Délimité par :
  (* ... *)
- Contient :
    - nom du module
    - description
    - variante ou version

### 2. Sections structurées

Les sections doivent être identifiables avec des commentaires structurants.

Formats acceptés :

- Niveau 1 :
  (* ========= TEXTE ========= *)

- Niveau 2 :
  (*---------- TEXTE ----------*)

- Fin de module :
  (*########## FIN *)

IMPORTANT :
- les séparateurs doivent être visibles (====, ----, #####)
- les commentaires doivent être fermés correctement

### 3. Commentaires de ligne

- Utilisés pour expliquer une logique non triviale
- Non obligatoires partout mais attendus sur les parties complexes

---

### 3. supérieur, inferieur ou égale
- Il ne faut pas qu'il y ai de signe supérieur "<" ou inférieur ">", uniquement supérieur ou égale "<=" ou bien inferieur ou égale ">="
